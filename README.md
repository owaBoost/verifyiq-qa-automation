# VerifyIQ QA Automation

AI-powered QA pipeline for VerifyIQ — generates and runs test cases against PR preview environments using Claude Code CLI.

## How It Works

1. Dev posts a PR number + preview URL in Slack
2. Open Claude Code in this folder and paste the QA prompt with the PR details
3. Claude fetches the PR diff, ClickUp ACs, and GCS fixtures automatically
4. Claude generates test cases, runs them, and posts results to ClickUp

## One-Time Setup

```bash
git clone https://github.com/owaBoost/verifyiq-qa-automation.git
cd verifyiq-qa-automation
npm install
cp .env.example .env
# Fill in .env with your actual values — see Environment Variables below
```

## Running QA for Any PR

Open Claude Code in this folder:

```bash
cd E:\verifyiq-qa-automation
claude
```

Then paste this prompt with the 3 values filled in from Slack:

```
You are a QA automation engineer for VerifyIQ. Run the full QA pipeline:

PR Number: [PR_NUMBER] from boost-capital/ai-parser-studio
Preview URL: [PREVIEW_URL]
ClickUp Tasks: [TASK_IDS] (comma separated)

Steps:
1. gh pr diff [PR_NUMBER] --repo boost-capital/ai-parser-studio
2. Fetch each ClickUp task: curl -H "Authorization: Bearer $CLICKUP_API_TOKEN" https://api.clickup.com/api/v2/task/[TASK_ID]
3. gsutil ls -r gs://qa-automation-dev/**
4. Assess if the PR is testable via API. If not, explain why and recommend another approach.
5. Generate test cases covering ALL AC scenarios — no count limit. Save to test-cases.json
6. Generate IAP token using E:/verifyiq-playwright/scripts/gen-iap-token.js with audience [PREVIEW_URL]
7. Run: node run_qa.mjs
8. Report results and post to ClickUp folder 90147709410
```

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `VERIFYIQ_API_KEY` | yes | VerifyIQ tenant API key |
| `CLICKUP_API_TOKEN` | yes | ClickUp personal API token |
| `GOOGLE_SA_KEY_FILE` | yes | Path to GCP service account JSON (e.g. `C:\Users\Admin\Downloads\qa-api-tester-key.json`) |
| `CLICKUP_FOLDER_ID` | yes | ClickUp folder for QA results (default: `90147709410`) |
| `GH_TOKEN` | yes | GitHub PAT with repo read access |
| `IAP_TOKEN` | optional | GCP IAP token — auto-generated by Claude Code |

## What Gets Created in ClickUp

- A **new list** per PR (e.g. `PR #284 - Fix Completeness Score v2`)
- One **task per test case** with pass/fail status and `qa-auto` tag
- Task **description** includes endpoint, payload, fixture path, and curl command
- Test **result posted as a comment** in the activity feed (not in description)
- **Summary comment** posted on the original ClickUp task(s) from the PR

## Files

| File | Purpose |
|---|---|
| `run_qa.mjs` | Test runner — executes TCs against preview env, posts results to ClickUp |
| `QA_PROMPT_TEMPLATE.md` | Full prompt reference with all rules, fixtures, and known fileTypes |
| `qa-run.sh` | Shell wrapper — loads `.env` and runs `run_qa.mjs` |
| `.env.example` | Template for environment variables |
| `package.json` | Node.js config (axios dependency) |
