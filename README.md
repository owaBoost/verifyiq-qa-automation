# VerifyIQ QA Automation

AI-powered QA pipeline — generates and runs test cases against PR preview environments using Claude Code CLI.

## How It Works

1. Dev posts a PR number + preview URL in Slack
2. Open Claude Code in this folder
3. Paste the QA prompt with PR details filled in
4. Claude fetches the diff, ClickUp ACs, and GCS fixtures — generates TCs, runs them, posts results to ClickUp

## One-time Setup

```bash
git clone https://github.com/owaBoost/verifyiq-qa-automation.git
cd verifyiq-qa-automation
npm install
cp .env.example .env
# Fill in .env with your actual values — see Environment Variables below
```

## Running QA for any PR

Open Claude Code in this folder:

```bash
cd E:\verifyiq-qa-automation
claude
```

Then paste this prompt with the 3 values filled in:

```
You are a QA automation engineer for VerifyIQ. Run the full QA pipeline:

PR Number: [PR_NUMBER] from boost-capital/ai-parser-studio
Preview URL: [PREVIEW_URL]
ClickUp Tasks: [TASK_IDS] (comma separated)

Steps:
1. gh pr diff [PR_NUMBER] --repo boost-capital/ai-parser-studio
2. For each ClickUp task: curl -H "Authorization: Bearer $CLICKUP_API_TOKEN" https://api.clickup.com/api/v2/task/[TASK_ID]
3. gsutil ls -r gs://qa-automation-dev/**
4. Assess if the PR is testable via API. If not, explain why and recommend another approach.
5. Generate test cases covering ALL AC scenarios — no count limit. Save to test-cases.json
6. Generate IAP token: node E:/verifyiq-playwright/scripts/gen-iap-token.js with audience [PREVIEW_URL]
7. cd E:\verifyiq-qa-automation && node run_qa.mjs
8. Report results and post to ClickUp folder 90147709410
```

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `VERIFYIQ_API_KEY` | yes | VerifyIQ tenant API key |
| `CLICKUP_API_TOKEN` | yes | ClickUp API token |
| `GH_TOKEN` | yes | GitHub PAT with repo permissions |
| `GOOGLE_SA_KEY_FILE` | yes | Path to GCP service account JSON key |
| `CLICKUP_FOLDER_ID` | yes | ClickUp folder ID (use `90147709410`) |
| `IAP_TOKEN` | optional | GCP IAP token — auto-generated by Claude Code |

## What Gets Created in ClickUp

For each PR run, the automation creates:

- A **new ClickUp list** named after the PR
- One **task per test case** with pass/fail status, payload, and curl command in description
- **Result comment** on each task with HTTP status, assertion details, and full API response
- **Summary comment** on the original ClickUp tasks linked in the PR

## Files

| File | Purpose |
|---|---|
| `run_qa.mjs` | Test runner — executes TCs, posts results to ClickUp |
| `QA_PROMPT_TEMPLATE.md` | Full prompt with all rules, fixture profiles, and known fileTypes |
| `qa-run.sh` | Shell wrapper — loads `.env` and runs `run_qa.mjs` |
| `.env.example` | Template for environment variables |
| `package.json` | Node.js dependencies (axios) |

## Test Case Schema

```json
{
  "test_cases": [
    {
      "id": "TC-01",
      "title": "Test name",
      "type": "positive",
      "endpoint": "/v1/documents/parse",
      "method": "POST",
      "payload": { "file": "gs://qa-automation-dev/...", "fileType": "BankStatement" },
      "expected_status": 200,
      "assertions": [
        { "description": "Check field", "path": "field.*.subfield", "pattern": "^regex$" }
      ]
    }
  ]
}
```

## Notes

- Always use wildcard paths `array.*.field` — never numeric index `array.0.field`
- Fixtures live in `gs://qa-automation-dev/` — Claude lists them automatically
- IAP token is required for `/ai-gateway/` endpoints — Claude generates it automatically
- `test-cases.json` is gitignored — generated fresh per PR run
